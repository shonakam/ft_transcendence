# transcendence-backend

このプロジェクトは、**Domain-Driven Design (DDD)** と **クリーンアーキテクチャ** の思想に基づいて設計されたNode.jsバックエンドアプリケーションです。

アーキテクチャは、関心の分離、高いメンテナンス性、テストの容易性を重視しており、ビジネスロジックをフレームワークやインフラストラクチャから独立させることを目的としています。

-----

## 🚀 思想 (Architectural Concepts)

本プロジェクトは、以下の4つの主要なレイヤーで構成されています。依存関係は常に外側から内側（`Interface` → `Usecase` → `Domain`）へと向かいます。

  * **`Domain` (ドメイン層)**
    アプリケーションの心臓部です。ビジネス上のルール、エンティティ（例: `User`）、値オブジェクト（VO）など、システムの核となるロジックを格納します。このレイヤーは、他のどのレイヤーにも依存しません。

  * **`Usecase` (ユースケース層)**
    アプリケーションが何をするのかを定義します。ドメイン層のオブジェクトを組み合わせて、具体的なユースケース（例: 「ユーザーを登録する」）を実現します。

  * **`Interface` (インターフェース層)**
    外部との接点を担当します。コントローラー、ルーター、WebSocketゲートウェイなどが含まれ、外部からのリクエストをユースケース層に渡す役割を担います。

  * **`Infra` (インフラストラクチャ層)**
    データベース接続、外部API連携など、具体的な技術の詳細を実装します。ドメイン層で定義されたリポジトリのインターフェースを、この層で実装します。

## 📂 ディレクトリ構成 (Directory Structure)

```plaintext
.
├── domain/         # ビジネスロジックのコア
│   ├── chat/
│   ├── game/
│   └── user/
│       ├── entity/     # エンティティ (Userなど)
│       ├── form/       # 入力フォームの定義
│       ├── repository/ # データ永続化のインターフェース
│       └── vo/         # 値オブジェクト (Email, Passwordなど)
│
├── usecase/        # アプリケーション固有のビジネスルール
│
├── interface/      # 外部との接点 (フレームワーク層)
│   ├── controller/ # リクエストを処理し、ユースケースを呼び出す
│   ├── router/     # URLとコントローラーをマッピング
│   └── websocket/  # WebSocket通信のゲートウェイ
│
├── infra/          # データベース、外部APIなどの具体的な実装
│
└── main.ts         # アプリケーションのエントリーポイント
```
